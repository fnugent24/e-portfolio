What is an ontology?

What do you understand about the ontology that has been presented for your reading this week? Could you attempt to define an ontology that would be relevant to the system that you are designing for the summative assessment?

According to Arnaut et al. (2010), an ontology, or ontologies, are defined as ‘explicit formal specifications of terms in a domain and their relationship to other terms, containing a set of concepts (entity, objects, domains, processes, goals, and results), properties, relationships, restrictions, or axioms.’ In other words, an ontology is a way to represent and organise information in a structured format (e.g. like a detailed catalogue) and to show how the concepts and categories of this specific subject area relate to one another.

In terms of attempting to define an ontology relevant to the system for the summative assessment, since our system is based on an online retail system, the ontology model would define various categories and subcategories that would help to organise the information (i.e. product items) in the system in a more structured way. Furthermore, since our application is specifically catered for selling plant seeds and gardening accessories, the various categories defined might be “plant seeds” with its subcategories being “flower seeds”, “vegetable seeds” etc., and “gardening accessories” with its subcategories being “garden tools”, “garden furniture”, “garden decorations” etc. The subcategories would help to show the relationship between the various products. Additionally, the categories/subcategories would include attributes such as price range, brand, material, size. The ontology can help to manage user inventory and notify about stock levels.

Referencs:

Arnaut, W., Oliveira, K. & Lima, F. (2010) OWL-SOA: A service oriented architecture ontology useful during development time and independent from implementation technology. 2010 Fourth International Conference on Research Challenges in Information Science (RCIS), pp. 523-532.

Codio activity:

Create a CLI/ shell that implements the following:

  • When you enter the command LIST it lists the contents of the current directory
  • The ADD command will add the following two numbers together and provide the result
  • The HELP command provides a list of commands available
  • The EXIT command exits the shell

Here is a CLI shell, as adapted from Praka (2018) and Szabo’s (2018) guides on writing shells in Python:


import sys

def main():
    while True:
        # Display a prompt and get user input
        user_input = input("Enter a command: ")

        # Check if the user wants to exit
        if user_input == 'exit' or user_input == 'EXIT':
            print("Goodbye!")
            break

        # execute commands
        if user_input == 'list':
            list_files()
        elif user_input == 'add':
            add_numbers()
        elif user_input == 'help':
            display_help()
        else:
            print("Invalid command. Type 'help' for a list of commands or 'exit' to exit application.")

def list_files():
    # Implement the 'list' command to list files in the current directory
    import os
    files = os.listdir('.')
    for file in files:
        print(file)

def add_numbers():
    # Implement the 'add' command to add two numbers entered by the user
    try:
        num1 = int(input("Enter the first number: "))
        num2 = int(input("Enter the second number: "))
        result = num1 + num2
        print(f"Result: {result}")
    except ValueError:
        print("Invalid input. Please enter valid numbers.")

def display_help():
    # Implement the 'help' command to display a list of available commands
    print("Available commands:")
    print("list - List files in the current directory")
    print("add - Add two numbers together")
    print("help - Display this help message")
    print("exit - Exit the application")

if __name__ == "__main__":
    main()

What are the two main security vulnerabilities with your shell?

One of the main security vulnerabilities associated with the shell is a lack of user authentication – the above code assumes that anyone who uses the shell can execute any command. This means that it is possible for hackers to use the application without any restrictions and can lead to them being able to access any sensitive data.

Another major security vulnerability with the above code is that of command injection (OWASP, n.d.) – this occurs by the attacker inputting potentially harmful commands to manipulate the system which can lead to system compromise, loss of sensitive and/or important data, and disruptions of system services.

What is one recommendation you would make to increase the security of the shell?

According to OWASP (2021), one way to increase the security of the shell (and, more specifically, prevent command injection vulnerabilities) would be to use safe and secure APIs. Not only does it increase security, but also the reliability and efficiency of the system.

Developing an API for a Distributed Environment:


from flask import Flask
from flask_restful import Api, Resource, reqparse
 
app = Flask(__name__)
api = Api(app)
 
users = [
    {
        "name": "James",
        "age": 30,
        "occupation": "Network Engineer"
    },
    {
        "name": "Ann",
        "age": 32,
        "occupation": "Doctor"
    },
    {
        "name": "Jason",
        "age": 22,
        "occupation": "Web Developer"
    }
]
 
class User(Resource):
    def get(self, name):
        for user in users:
            if(name == user["name"]):
                return user, 200
        return "User not found", 404
 
    def post(self, name):
        parser = reqparse.RequestParser()
        parser.add_argument("age")
        parser.add_argument("occupation")
        args = parser.parse_args()
 
        for user in users:
            if(name == user["name"]):
                return "User with name {} already exists".format(name), 400
 
        user = {
            "name": name,
            "age": args["age"],
            "occupation": args["occupation"]
        }
        users.append(user)
        return user, 201
 
    def put(self, name):
        parser = reqparse.RequestParser()
        parser.add_argument("age")
        parser.add_argument("occupation")
        args = parser.parse_args()
 
        for user in users:
            if(name == user["name"]):
                user["age"] = args["age"]
                user["occupation"] = args["occupation"]
                return user, 200
        
        user = {
            "name": name,
            "age": args["age"],
            "occupation": args["occupation"]
        }
        users.append(user)
        return user, 201
 
    def delete(self, name):
        global users
        users = [user for user in users if user["name"] != name]
        return "{} is deleted.".format(name), 200
      
api.add_resource(User, "/user/")
 
app.run(debug=True)
