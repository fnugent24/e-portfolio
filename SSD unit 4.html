Secure Software Development Unit 4

Programming Language Concepts

1. What is ReDOS and what part do ‘Evil Regex’ play?

According to Weidman (n.d.), a Regular expression Denial of Service (ReDoS) is a type of cybersecurity attack (more specifically, a Denial of Service attack). This attack exploits Regular Expression implementations that are inefficient or poorly designed which causes them to work very slowly or become unresponsive. “Evil Regex” patterns play a crucial role in the context of ReDoS attacks. Weigman (n.d.) states that ‘A Regex pattern is called Evil Regex if it can get stuck on crafted input’. These patterns are specially generated input strings that are designed to ‘expose common errors made by programmers’ in order to evaluate the performance of regular expressions and to observe and rectify potential security vulnerabilities (Larson and Kirk, 2016).

2. What are the common problems associated with the use of regex? How can these be mitigated?

One common problem associated with the use of regex is that its nuance can make it prone to errors and bugs. For example, as noted by Larson (2018), while a failure will arise with unbalanced parentheses, it will not with unbalanced braces []. Furthermore, some symbols have distinct meanings based on their situation. This makes it difficult for developers to understand the rules and can be easy for them to make mistakes during development. An additional problem, mentioned above, is that poorly designed regular expressions can be inefficient and slow, particularly when applied to large input texts. Another problem relates to issue of false positives and negatives. Regular expression patterns may inadvertently match or fail to match certain inputs, leading to false positives (matching when it shouldn’t – i.e. accepting strings that should be rejected) or false negatives (not matching when it should – i.e. rejecting strings that are acceptable). These mistakes can cause a program to operate incorrectly or potentially crash (Larson, 2018). To mitigate the problems outlined here, it is important to use regular expressions in the right situation and where they are most suitable for the task at hand. Furthermore, developers should follow best practices when using regular expressions, including writing clear and efficient patterns, testing them thoroughly, and documenting them appropriately.

3. How and why could regex be used as part of a security solution?

Regular expressions (regex) can be used as part of a security solution in various ways to improve the security of software systems and applications. One way is that regex can help in defining authentication mechanisms. For example, they can be used to match and validate user credentials or identify access permissions based on patterns in usernames or roles. Regex can also be used to validate user input (such as email addresses, phone numbers, etc.) by confirming whether the data entered by users complies with the expected formats. By doing this, they help prevent security vulnerabilities such as injection attacks.
