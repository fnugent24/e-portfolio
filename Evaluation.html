Evaluation of Final Project vs. Design Proposal

Evaluation of Final Project vs. Design Proposal

Your evaluation of the final project (Unit 11) vs. the design proposal (Unit 6), highlighting any differences with justifications for any changes/ omissions.

The key differences between the design documentation and the actual coding output developed were that:

  • We replaced the API technology/web framework utilised from Flask to FastAPI. This is because, although Flask is a lightweight and flexible microframework, FastAPI has a lower ‘ease of first time use’ barrier and more importantly, it provides the user with a user interface via http or https if SSL certification is turned on via the application back-end.

  • The Mccabe linter was not utilised, instead Pylint and Flake8 were used. We felt that Pylint and Flake8 would provide sufficient linting. The Pylint linter was used to identify stylistic, syntactical and programming errors and implement their solutions, while the Flake8 linter was used to prevent things like syntax errors, typos, bad formatting and incorrect styling.
