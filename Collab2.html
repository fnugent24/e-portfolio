Collaboration Discussion Activity Part 2

Collaborative Discussion 2:

  • The (anonymous) TrueCrypt authors have said “Using TrueCrypt is not secure as it may contain unfixed security issues” (TrueCrypt, 2014). Does the cryptanalysis provided above prove or disprove this assumption?
  • Would you be prepared to recommend TrueCrypt to a friend as a secure storage environment? What caveats (if any) would you add?
  • Present an ontology design which captures the weaknesses of TrueCrypt, and organise them according to their severity. Expand the ontology design by considering the factors which will cause each weakness to become an issue from a user's perspective. For example, if a user wishes to encrypt a disk storing bank details using TrueCrypt, which weakness of the software might cause this specific user goal to be negatively impacted?

    • The cryptanalysis report provided by Junestam and Guigo (2014) does support the assumption by the authors of TrueCrypt that “[u]sing TrueCrypt is not secure as it may contain unfixed security issues” (TrueCrypt, 2014). The findings of the report reveal four cryptography vulnerabilities in TrueCrypt, indicating that TrueCrypt lacks a fundamental sense of security. Furthermore, the report highlights these issues as potentially leading to unpredictable results, whereby it is stated that “CryptAcquireContext may silently fail in unusual scenarios” as well as “AES implementation [being] susceptible to cache-timing attacks” (Junestam and Guigo, 2014; p, 12), where both of these are marked as high in severity levels. Thus, the findings support the assumption by the authors of TrueCrypt that the application “may contain unfixed security issues”. On the other hand, as noted by cryptography specialist Matt Green, who commented on the report, “[t]he NCC audit found no evidence of deliberate backdoors, or any severe design flaws that will make the software insecure in most instances.” (Green, 2015). This is also important to bear in mind as the findings did not explicitly declare TrueCrypt as being insecure piece of software. However, the undoubtable thing is that TrueCrypt is no longer maintained and has not been since 2014. This means that if a security vulnerability is found, there will not be anybody to report it to, which means that it will not ever be fixed. Thus, while it may be deemed as a secure system now, there is an ever-growing risk that it may become insecure sometime in the future.

    In relation to the question of whether I would recommend TrueCrypt as a secure storage environment, I would again point to what I wrote above about the lack of maintainability of TrueCrypt by the developers, which would inevitably lead it to becoming more at risk of security vulnerabilities over time. Furthermore, as the nature of TrueCrypt is that it is inherently linked to the encryption of data from storage devices, I would be hesitant to ultimately be prepared recommend it to others.

    Peer Response 1:

    Hi Liam, I agree on the whole with your argument that the cryptanalysis report provided by Junestam and Guigo (2014) supports the assumption that “[u]sing TrueCrypt is not secure as it may contain unfixed security issues” (TrueCrypt, 2014). A few points which I'm sure you were aware of but that I thought would still be helpful to include in this discussion are:

  • That TrueCrypt has not been maintained/updated by the developers since 2014. This means that if a security vulnerability is found, there will not be anybody to report it to, and thus the issue(s) will not be able to be fixed.

  • It's important to note that while we can assume from the cryptoanalysis report that using TrueCrypt is potentially unsafe, the report does not explicitly state that the errors found make TrueCrypt a fundamentally insecure piece of software. This is supported by cryptography specialist Matt Green, who comments that “[t]he NCC audit found no evidence of deliberate backdoors, or any severe design flaws that will make the software insecure in most instances.” (Green, 2015).

    • In addition to your point about not being prepared to recommend TrueCrypt to a friend, it might perhaps as well be useful to suggest some alternative applications to use rather than TrueCrypt. From this, it might be worth going into more detail about the possible applications out there and what benefits they would bring to the user. An option could be to suggest the use of VeraCrypt, which is the direct successor of TrueCrypt and which fixed many of the security issues associated with TrueCrypt, and continues to be maintained by the developers (VeraCrypt, n.d.)
